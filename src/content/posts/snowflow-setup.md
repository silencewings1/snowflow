很多人搭个人站，第一反应是"开个技术博客"。我一开始也是这么想的，但做到一半发现方向拧了——我要的不是写文章的地方，而是放项目的地方。这篇记录这个定位转变和最终的技术选型。

## 一开始的需求

最初定的几条硬约束：

| 维度 | 选择 |
|------|------|
| 技术栈 | Vue / React + Vite 系 |
| 定位 | 技术分享博客 |
| 视觉风格 | 现代设计感 |
| 托管 | 自托管 VPS |
| 集成形态 | 独立子站，可挂 todayfood |

纯 SPA 做博客有三个硬伤：SEO 差、写作不友好、首屏慢。所以方向很快收敛到 SSG（静态站点生成器），最终框架定了 **Astro**——底层就是 Vite，原生支持 Vue/Svelte 组件，SSG 默认，SEO 强。

## 主题的纠结

框架定了，主题反而纠结最久。一开始相中了 **Fuwari**（saicaca/fuwari），理由是：

- 现代设计感拉满：封面大图、卡片流、毛玻璃、暗色模式
- 功能完整：文章列表 / 标签 / 归档 / 关于页，开箱即用
- 技术栈现代：Astro + Svelte + Tailwind + TypeScript

但搭起来后我犯了个错：被"技术博客"这个定位绑架，去搜了一堆极简主题（AstroPaper、Cactus、Nano）。理由是技术博客不该有封面图焦虑、不该有华丽动效。

## 关键的认知转弯

后来想明白了：**我根本不是要写技术博客，我是要展示项目。**

- 我有 todayfood 这样的实际项目要挂
- 我完全可以接受首页大图，甚至想要二次元调性
- 文章是次要的，项目卡片才是主角

Fuwari 的调性（二次元、大图、作品集气质）正好对路。极简主题反而是错的方向。

## 最终方案

保留 Fuwari，但做了几件事把"博客"扭成"项目集"：

1. **新增 `/projects` 页**：卡片网格展示项目，整卡指向项目外链
2. **导航重排**：主页 → 项目 → 关于 → 归档 → GitHub，把项目提到第一梯队，归档降级
3. **弱化文章流**：文章退居二线，主要写项目相关的复盘和决策记录
4. **备案号挂 Footer**：站点定位为项目集主页，一个域名一个站，备案号挂一次即可

## 关于一个细节：Fuwari 用的是 Svelte 不是 Vue

选型文档里原本写"Fuwari 是 Astro + Vue"，实际拉下来发现用的是 Svelte。这不影响核心结论（Vite 底层 + SSG + SEO + 现代设计感都还在），只是交互组件框架从 Vue 变成 Svelte。如果强求 Vue 栈，得换主题或自己改造，但我能接受 Svelte。

## 小结

最大的教训是：**选主题前先想清楚站点定位**。"技术博客"和"项目集"是两种完全不同的东西，前者重阅读体验要极简，后者重视觉冲击要华丽。定位错了，主题选得再认真也是南辕北辙。
